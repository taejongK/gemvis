# [ARCHIVED] Gemma 4 E2B 로컬 서빙 테스트 가이드 (Windows)

> **⚠️ ARCHIVED**: 이 문서는 Ollama vs llama.cpp 비교 테스트의 역사적 기록입니다.  
> **현재 프로젝트**: llama.cpp만 사용 (GPU 가능 시 GPU, 아닐 시 CPU).  
> **참고용으로만 보존됩니다.**

## 목적

Gemvis의 AI 분석을 클라우드 API 없이 로컬 LLM으로 대체할 수 있는지 Windows 환경에서 검증한다.
`gemma-4-E2B-it-Q4_K_M` (3.46GB, 4bit 양자화)을 ollama와 llama.cpp 두 가지 방식으로 서빙하고 성능을 비교한다.

**결론**: llama.cpp 채택 (GPU/CPU 유연성, 작은 모델 크기, Unsloth GGUF 호환)

## 테스트 환경

| 항목 | 스펙 |
|------|------|
| CPU | Intel i9-12900 (24 threads) |
| RAM | 16GB DDR5 |
| GPU | NVIDIA RTX 3060 12GB VRAM |
| OS | Windows 11 |
| 모델 | gemma-4-E2B-it-Q4_K_M (3.46GB) |

## 모델 정보

- **Gemma 4 E2B**: Google의 Edge 모델, effective 2.3B / 전체 5.1B 파라미터
- **Q4_K_M**: 4bit 양자화, 품질-크기 밸런스 최적 (추천 양자화)
- **VRAM 요구**: ~4GB (RTX 3060 12GB에 여유롭게 적재 가능)
- 멀티모달 지원: 이미지, 오디오 입력 가능

## Part 1: Ollama 테스트

> llama.cpp를 래핑한 올인원 패키지. 설치부터 서빙까지 가장 간단하다.

### 1-1. 설치

1. https://ollama.com/download/windows 에서 `OllamaSetup.exe` 다운로드
2. 설치 프로그램 실행 (관리자 권한 불필요, 사용자 폴더에 설치됨)
3. 설치 경로: `%LOCALAPPDATA%\Programs\Ollama`
4. 설치 후 시스템 트레이에 ollama 아이콘 표시됨
5. PowerShell 또는 CMD에서 확인:

```powershell
ollama --version
```

### 1-2. Unsloth 모델 다운로드

HuggingFace에서 Unsloth가 배포한 Gemma 4 E2B GGUF 시도

```powershell
ollama run hf.co/unsloth/gemma-4-E2B-it-GGUF:Q4_K_M
```

### 1-3. 500 에러 발생

다운로드는 성공하지만 모델 로드 단계에서 다음 에러가 발생한다:

```
pulling manifest
pulling ac0069ebccd3: 100% ▕████...▏ 3.1 GB
pulling 36510d06fc72: 100% ▕████...▏ 985 MB
...
verifying sha256 digest
writing manifest
success
Error: 500 Internal Server Error: unable to load model: ...\blobs\sha256-ac0069ebccd3...
```

**원인:**
- Unsloth가 양자화한 Gemma 4 GGUF는 **텍스트 모델과 비전 모델이 분리된(split) 형태**로 배포된다 (다운로드 시 blob이 2개 생기는 이유).
- Split 모델은 llama.cpp 백엔드에서만 실행 가능한데, **llama.cpp는 아직 Gemma 4 아키텍처를 공식 지원하지 않는다**.
- Ollama는 다음 벤더 싱크(vendor sync)에서 llama.cpp의 Gemma 4 지원을 병합할 예정이므로, 그 전까지는 Unsloth 모델이 동작하지 않는다.

### 1-4. 대체 방법 (1): Unsloth 모델 워크어라운드 (vision 블롭 제거)

Unsloth GGUF를 굳이 써야 한다면, **vision 블롭을 제외하고 text-only로 로드**하도록 Modelfile을 수정한다.

**1) 기존 Modelfile 추출**

```powershell
ollama show --modelfile hf.co/unsloth/gemma-4-E2B-it-GGUF:Q4_K_M | Out-File -Encoding utf8 gemma_4_unsloth_modelfile
```

**2) 두 번째 FROM 라인 주석 처리**

메모장 또는 VS Code로 열어서 편집:

```powershell
notepad gemma_4_unsloth_modelfile
```

`FROM` 라인 2개 중 두 번째(vision 모델 blob)를 `#`으로 주석 처리:

```modelfile
# Modelfile generated by "ollama show"
# To build a new Modelfile based on this, replace FROM with:
# FROM hf.co/unsloth/gemma-4-E2B-it-GGUF:Q4_K_M

FROM C:\Users\<사용자>\.ollama\models\blobs\sha256-<text-model-hash>
#FROM C:\Users\<사용자>\.ollama\models\blobs\sha256-<vision-model-hash>   ← 이 줄 주석 처리
TEMPLATE "{{ if .System }}<bos><|turn>system
{{ .System }}<turn|>
{{ end }}{{ if .Prompt }}<|turn>user
{{ .Prompt }}<turn|>
{{ end }}<|turn>model
{{ .Response }}<turn|>
"
PARAMETER stop <bos>
PARAMETER stop <|turn>
PARAMETER stop <turn|>
PARAMETER stop <|turn>user
```

**3) 새 모델로 등록 & 실행**

```powershell
ollama create gemma-4-unsloth -f gemma_4_unsloth_modelfile
ollama run gemma-4-unsloth
```

>### 워크어라운드의 한계: 멀티모달 불가
>
>vision 블롭을 주석 처리했기 때문에:
>- ✅ 텍스트 대화, JSON 출력 등 텍스트 기반 작업은 정상 동작
>- ❌ **이미지 입력이 완전히 불가능** (비전 인코더가 로드되지 않음)
>
>**Gemvis 관점에서 문제점:**
>Gemvis는 파일 분석 기능에 **이미지 파일(JPG, PNG 등) 분석도 포함**한다. 가족 사진에서 사람, 장소, 이벤트를 추출하려면 비전 기능이 필수인데, 이 워크어라운드로는 사진 분석이 불가능하다.
>
>→ Unsloth 워크어라운드는 **텍스트 전용 용도** (예: 코드 분석, 텍스트 요약)에만 쓸 수 있다.
>
### 1-5. 대체 방법 (2): Ollama 공식 양자화 모델 사용

Unsloth의 500 에러와 워크어라운드의 멀티모달 손실 문제를 모두 피하려면, **Ollama가 자체 양자화한 공식 모델**을 쓴다.

```powershell
# 공식 ollama 레지스트리 모델 (권장)
ollama run gemma4:e2b-it-q4_K_M
```

**Unsloth vs Ollama 공식 비교**

| 항목 | `hf.co/unsloth/...` | `gemma4:e2b-it-q4_K_M` |
|------|---------------------|------------------------|
| 배포자 | Unsloth (서드파티) | **Ollama 공식** |
| 호스팅 | HuggingFace | ollama.com 레지스트리 |
| 양자화 | Unsloth Dynamic (UD) | Ollama 자체 양자화 |
| Split GGUF | 예 (text + vision 분리) | 아니오 (통합) |
| 현재 동작 (2026-04) | ❌ 500 에러 | ✅ 바로 동작 |
| 멀티모달 (이미지) | 워크어라운드 시 텍스트만 | ✅ 지원 |

**결론:** Gemvis처럼 이미지 분석이 필요한 용도에서는 **`gemma4:e2b-it-q4_K_M`가 최적**. Unsloth UD 양자화 품질 이점은 llama.cpp의 Gemma 4 정식 지원 이후로 미룬다.

### 1-6. OpenAI 호환 API 서버로 사용

ollama는 기본적으로 `localhost:11434`에 OpenAI 호환 API를 제공한다.

```powershell
# PowerShell에서 API 호출
$response = Invoke-RestMethod -Uri "http://localhost:11434/v1/chat/completions" `
  -Method POST -ContentType "application/json" `
  -Body '{"model":"gemma4:e2b-it-q4_K_M","messages":[{"role":"user","content":"안녕하세요"}]}'

# 응답 텍스트
$response.choices[0].message.content

# 토큰 사용량
$response.usage
```

## Part 2: llama.cpp 테스트

> C/C++ 네이티브 빌드. GPU 오프로드, 배치 크기 등 세밀한 튜닝 가능.

### 2-1. 설치

```powershell
winget install llama.cpp
```

### 2-2. 모델 다운로드 + 서버 실행 (한번에)

`-hf` 옵션으로 HuggingFace에서 자동 다운로드하면서 서버를 실행한다:

```powershell
llama-server -hf unsloth/gemma-4-E2B-it-GGUF:Q4_K_M `
  --port 8080 `
  -ngl 999 `
  --ctx-size 8192 `
  --flash-attn
```

주요 옵션:
- `-hf`: HuggingFace 저장소에서 자동 다운로드
- `-ngl 999`: 모든 레이어를 GPU에 오프로드
- `--ctx-size`: 컨텍스트 길이 (기본 4096, 최대 128K)
- `--flash-attn`: Flash Attention으로 VRAM 절약
- `-ngl 0`: CPU 전용 모드 (비교용)

모델 저장 위치: `%USERPROFILE%\.cache\huggingface\hub\`

### 2-3. API 테스트

llama-server는 OpenAI 호환 API를 `localhost:8080`에 제공한다.

```powershell
# PowerShell에서 API 호출
$response = Invoke-RestMethod -Uri "http://localhost:8080/v1/chat/completions" `
  -Method POST -ContentType "application/json" `
  -Body '{"model":"gemma-4-E2B-it","messages":[{"role":"user","content":"안녕하세요, 자기소개 해주세요."}]}'

$response.choices[0].message.content
```

웹 UI도 기본 제공: 브라우저에서 `http://localhost:8080` 접속.

### 2-4. CPU 전용 vs GPU 비교

```powershell
# CPU 전용 모드 (-ngl 0)
llama-server -hf unsloth/gemma-4-E2B-it-GGUF:Q4_K_M `
  --port 8080 `
  -ngl 0 `
  --ctx-size 8192

# 동일한 프롬프트로 테스트 후 tokens/sec 비교
```

### 설치 & 다운로드

| 항목 | Ollama (공식) | llama.cpp (Unsloth) |
|------|---------------|---------------------|
| 설치 시간 | 1분 40초 | (winget) 수분 이내 |
| 모델 다운로드 시간 | 1분 30초 | 1분 |

### 성능 (동일 프롬프트 기준)

| 항목 | Ollama (공식) | llama.cpp GPU (Unsloth) | llama.cpp CPU (Unsloth) |
|------|---------------|-------------------------|-------------------------|
| Cold start (첫 로드) | 5.59s | - | - |
| Prompt eval (tok/s) | 2478.53 | 71.96 (cold) / 192.46 (warm) | 12.08 |
| Generation (tok/s) | 113.48 | 98.18 | 15.81 |
| Total time (1회) | 19.84s (1518 tok) | 19.83s (1913 tok) | 137.23s (2139 tok) |
| VRAM 사용량 | | ~5.5GB (전체) | - |
| RAM 사용량 | | ~1.4GB | ~1.3GB |
| CPU 사용률 | | | ~65% |
| 응답 품질 (1-5) | 4 | 4 | 4 (속도 외 품질 동일) |

> 참고:
> - Ollama: `gemma4:e2b-it-q4_K_M` (공식 통합 GGUF, 6.67GB)
> - llama.cpp: `unsloth/gemma-4-E2B-it-GGUF:Q4_K_M` (Unsloth split GGUF, 3.81GB = 텍스트 2.89GB + 비전 0.92GB)
> - **주목:** Unsloth split GGUF는 Ollama에서는 500 에러가 나지만, llama-server로는 직접 실행 가능
> - Generation 속도: Ollama 113 tok/s > llama.cpp GPU 98 tok/s > llama.cpp CPU 16 tok/s
> - **GPU vs CPU**: Generation 약 **6배 차이** (98 → 16 tok/s), 전체 시간은 약 **7배 차이** (20s → 137s)
> - CPU 모드는 GPU 없는 환경(노트북 등)에서만 권장

## Part 5: llama.cpp 시나리오

> llama.cpp 기반으로 Gemvis를 구성할 때의 전체 흐름과 고려사항.

### 설치 & 실행 흐름

```
1. winget install llama.cpp
2. llama-server -hf unsloth/gemma-4-E2B-it-GGUF:Q4_K_M -ngl 999 --port 8080
   → 자동으로 HuggingFace에서 모델 다운로드 (3.81GB)
   → 서버가 localhost:8080에 기동
3. Gemvis에서 OPENAI_API_BASE=http://localhost:8080/v1 로 연결
4. 폴더 지정 후 파일 분석 시작
```

### 전체 소요 시간 측정

| 단계 | 소요 시간 | 비고 |
|------|-----------|------|
| llama.cpp 설치 (winget) | 수분 이내 | |
| 모델 다운로드 (3.81GB) | 약 1분 | `-hf` 플래그로 자동 다운로드, 네트워크 속도 의존 |
| 첫 모델 로드 (GPU) | ~수 초 | `-ngl 999` 기준 |
| 첫 응답 생성 (cold start) | ~20초 | 양자역학 설명 기준, 1913 tok 생성 |
| **총 소요 시간** | **약 2~3분** | 모델 다운로드가 대부분 |

### PC 최소 사양

| 항목 | 최소 | 권장 |
|------|------|------|
| CPU | x86_64 | i5/Ryzen 5 이상 |
| RAM | 8GB | 16GB |
| VRAM (GPU) | - (CPU 모드 가능하지만 6배 느림) | **6GB+** (RTX 3060 12GB 검증) |
| 디스크 여유 공간 | 5GB | 10GB+ |
| OS | Windows 10 | Windows 11 |

### llama.cpp 장단점 (Gemvis 관점)

**장점:**
- Unsloth split GGUF 정상 동작 (Ollama는 불가)
- 모델 크기 작음 (3.81GB vs Ollama 6.67GB)
- `--ctx-size`, `-ngl`, `--flash-attn` 등 세밀한 튜닝 가능
- 단일 실행 파일 → 번들링 용이

**단점:**
- Generation 속도 Ollama 대비 약 15% 느림 (98 vs 113 tok/s)
- 모델 관리 수동 (ollama pull/rm 같은 편의 기능 없음)
- 백그라운드 서비스화 직접 구현 필요 (Ollama는 트레이 아이콘 자동)

### Gemvis 통합 시나리오

| 시나리오 | LLM 엔진 | 장점 |
|---------|---------|------|
| **일반 사용자용** | Ollama | 원클릭 설치, 자동 백그라운드, 멀티모달 기본 지원 |
| **파워 유저/개발자용** | llama.cpp | 파일 크기 작음, 세밀한 튜닝, 모델 소스 유연성 |
| **GPU 없는 환경** | llama.cpp CPU 모드 | Ollama는 GPU 강제, llama.cpp는 `-ngl 0` 선택 가능 |

## Part 6: 비교 정리

### Ollama vs llama.cpp

| 관점 | Ollama | llama.cpp |
